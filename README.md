# barsdotnet_scheduler

3. Планировщик
Реализовать web-приложение планировщик
Требования:
* система регистрации/авторизации
* основной экран пользователя - экран задач, разделён на три горизонтальные части. Первая просроченные задачи, задачи срок исполнения которых прошёл. Вторая текущие задачи. Третья - будущие задачи. Интервалв котором задачи считаются текущими задаётся в настройках.
* возможность добавлять задачи (заголовок, текст, крайняя дата исполнения, интервал после которого задача будет считаться просроченной перекрывает интервал из настроек)
